# RFC Process

Use RFCs for language semantics, compiler/runtime architecture, public interfaces, and other design changes with long-term impact.

## Lifecycle
1. Create RFC from `rfcs/000-template.md`
2. Open PR marked `Draft`
3. Gather feedback and revise
4. Maintainers mark as `Accepted` or `Rejected`

## Numbering
Use zero-padded numbering: `000`, `001`, `002`, etc.

## Status Values
- `Draft`
- `Accepted`
- `Rejected`
- `Superseded`
